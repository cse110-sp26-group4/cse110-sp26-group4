# ADR-001: Build an Agent-Native Issue Tracker Rather Than a General Tracker with Agent Features

## Status
Accepted

## Context
The team needed to decide how AI agents would fit within the issue tracking system. We had originally been working towards a general-purpose issue tracker with AI agent capabilities built on top. The alternative was to create an issue tracker designed specifically to guide agent workflows. Our decision was shaped by the short 1 month time constraint.

## Decision
We will build an Issue Tracker for Agents, that is an issue tracking system designed around agent workflows, rather than an existing-style issue tracker with agent features added on top.

The narrower scope allows the team to:
- Work from existing reference examples and prior art
- Deliver a more focused, complete product within the available time
- Avoid the complexity of designing a general-purpose system alongside agent-specific requirements

## Consequences

### Easier
- Scope is well-defined and bounded to less features
- Existing implementations can be used as reference to guide design
- Team can work faster with fewer open-ended decisions

### More Difficult
- Tool has a smaller user market compared to human issue tracking tools
