# ADR-004: Raw SQLite strings vs Drizzle ORM


## Status
Accepted.

## Context
During the Professor meeting, Powell brought up flexibility as a concern. Specifically, he pointed out how if our schema changes it requires many changes in our subsequent files. Due to the way the better-sqlite package is, we have to keep our SQL queries in immutable strings, which isn't good. So, instead we can use an ORM (object relational mapper) which allows us to write JavaScript which then is translated into SQL queries.

## Decision
We propose using the Drizzle ORM for SQL queries.

## Consequences
What becomes easier or more difficult to do because of this change?

### Easier
- Allows us to write SQL queries in JS
- Let's us dynamically write our SQL tables with our schema models
- Makes queries easier to read and interact with
- You can still write raw SQL queries, so there is no functionality being lost

### Harder
- There is a learning curve to using an ORM, because the way you formulate SQL commands changes slightly
