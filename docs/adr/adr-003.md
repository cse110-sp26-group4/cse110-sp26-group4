# ADR-003: Use SQLite vs JSON for storage


## Status
Accepted.

## Context
We need to store the issues somehow. We can achieve this through either json files or a structured db like SQLite.


## Decision
We are using SQLite over JSON for the following reasons:
1. A structured db gives us better control over our data
2. An actual SQLite db allows us to have relationships between data and easy querying
3. JSON storage can get messy and out of control easily, also requires custom parsing

## Consequences
What becomes easier or more difficult to do because of this change?

This change makes it easier to work with our data for all the reasons listed above. This change might make things difficult from a development perspective because SQL is a new language for a lot of us. SQL strings and queries can be quite confusing so there is a learning curve there. Overall, we think these downsides are worth it for the power of SQL databases.
