## Notes
- Whose problem are we solving? (Who are our users)
  - Someone who uses a lot of AI agents
- What would a successful MVP accomplish?
  - Must Have:
    - AI needs a set of API CRUD (Create, Read, Update, Delete) endpoints
    - Full activity history so users can see every ticket change made by humans or AI
  - Should Have:
    - Some sort of CRUD attempt / resolution time limit per ticket, to prevent AI from spending too many tokens on attempting to solve a ticket without user supervision 
    - Human interactions with the issues alongside the AI
  - Could Have: 
    - Agents could be able to comment/report on issues they faced while completing a task
    - Web UI to see and interact with tasks
    - Agents can suggest future directions the project can go
    - Integrations with Github or Slack
  - Won’t Have:
    - The Agent deleting / replacing massive amounts of code without approval
    - Doesn’t need to be a pretty UI
    - No stats
    - Won’t support multiple users or teams
- What platform are we building on?
  - Terminal CLI / Command Line Tool
  - VSCode Extension
  - Website Dashboard
  - Electron Apps
  - Slack Extension
- Terminal CLI
  - Pros:
    - More support for building and using
    - Can use Beads as an example
  - Cons:
    - How to be creative / be different to Beads
    - Limited UI, not very user friendly
- VSCode Extension
  - Pros:
    - More visually appealing than a terminal CLI
  - Cons:
    - Less flexible than Terminal CLI

## Decisions Made
Issue Tracker for Agents vs. Issue Tracker w/ Agent features
- We are building an Issue Tracker for Agents
- Rationale: Smaller scope, reference examples, more doable
Our Platform:
- Terminal CLI vs VSCode Extension
- We chose Terminal CLI
- Rationale: Lowest complexity, more feasible with time remaining
