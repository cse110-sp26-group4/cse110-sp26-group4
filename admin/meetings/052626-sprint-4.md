## Convention
- Feature branching:
  - Each person should create a new branch for each specific task.
  - Once the feature is fully complete create a PR to the main branch which closes the issue (issues should be made prior to coding).
- Branch naming: : `<type>/<name>/<short-desc.>`
  - Ex: docs/zach/project-requirements
- Branch Types:
  - Feature (or Feat): for developing new functionality
  - Fix: for addressing bug fixes in existing features
  - Docs: for updates, additions, or corrections to project documentation
  - Refactor: For structural code changes that do not add features or fix bugs.
  - Test: For adding or modifying tests.
  - Experiment: For trying out new ideas or prototypes outside the main workflow.
- Make sure to close/delete branches that are completed to keep the repo clean!
