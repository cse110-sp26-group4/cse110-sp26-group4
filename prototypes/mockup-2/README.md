# Mockup 2.0: AI-Powered Issue Tracker

This document outlines the design for an AI-powered issue tracking system. This mockup expands on the ideas from `mockup-1` and the various workflows.

## Design Documents

*   [**CLI Design (`cli.md`)**](./cli.md): Describes the command-line interface for human users.
*   [**Agent API (`agent_api.md`)**](./agent_api.md): Details the CRUD API for AI agents to interact with issues.
*   [**Human UI (`human_ui.md`)**](./human_ui.md): Proposes a graphical user interface for human users to manage tickets.
*   [**Data Model (`data_model.md`)**](./data_model.md): Defines the data structures for tickets, users, agents, and other core entities.
*   [**Activity Log (`activity_log.md`)**](./activity_log.md): Explains the system for logging all actions taken by humans and AI.
*   [**Sprint Dashboard (`sprint_dashboard.md`)**](./sprint_dashboard.md): Lays out the design for a dashboard to track progress.
*   [**Chat & Communication (`chat.md`)**](./chat.md): Covers the general and per-ticket chat systems.
*   [**Token Management (`token_management.md`)**](./token_management.md): Describes the mechanism for tracking and controlling AI token usage.
*   [**Agent Accountability (`agent_accountability.md`)**](./agent_accountability.md): Details the system for monitoring and managing AI agent performance.
*   [**Ticket Lifecycle & Dependencies (`ticket_lifecycle.md`)**](./ticket_lifecycle.md): Outlines the rules for ticket resolution, time limits, and handling dependencies.
*   [**Approval Workflows (`approval_workflows.md`)**](./approval_workflows.md): Describes the process for human approval of AI-proposed changes.
